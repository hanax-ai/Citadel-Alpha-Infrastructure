## Template Usage Instructions

**How to use this template:**

1. **Fill in all placeholders** in square brackets with actual values
2. **Remove unused sections** if they don't apply to your test case
3. **Add additional steps** as needed in the Test Steps section
4. **Update status fields** as the test progresses through execution
5. **Maintain version history** for all changes to the test case
6. **Link to related documents** and requirements for traceability

**Best Practices:**
- Keep test steps atomic and verifiable
- Use clear, unambiguous language
- Include all necessary verification points
- Document both positive and negative test scenarios
- Ensure test case is maintainable and reusable

**Status Guidelines:**
- **Not Started**: Test case created but not yet executed
- **In Progress**: Test execution is underway
- **Passed**: All pass criteria met successfully
- **Failed**: One or more fail criteria triggered
- **Blocked**: Cannot execute due to blocking issues
- **Skipped**: Intentionally not executed in current cycle

---

*This test case template provides a comprehensive framework for documenting, executing, and tracking individual test cases with full traceability and detailed results.*
